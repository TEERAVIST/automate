Permission Management Script Guide

Service Permissions Java Directory General Setup Config Files Symbolic Links

Set Service Permissions

Use set_service_permissions to assign strict permissions to service files. 

set_service_permissions "/path/to/service/file"
Example:

set_service_permissions "/etc/systemd/system/my-service.service"

Sets permissions to 640 and ownership to root:root.

Set Java Directory Permissions

Use set_java_permissions to ensure ownership and permissions for Java directories. 

set_java_permissions "/path/to/java" "user:group"
Example:

set_java_permissions "/opt/java" "java_user:java_group"

Sets ownership to java_user:java_group and permissions to 755.

Setup General Permissions

Use setup_permissions to recursively set permissions and ownership for a path. 

setup_permissions "/path/to/directory" "user:group" 775
Example:

setup_permissions "/var/www/myapp" "www-data:www-data" 775

Sets permissions for the path to 775 and ownership to www-data:www-data.

Set Configuration File Permissions

Use set_config_permissions to apply strict permissions to configuration files. 

set_config_permissions "/path/to/config/file"
Example:

set_config_permissions "/etc/myapp.conf"

Sets permissions to 600 and ownership to root:root.

Set Symbolic Link Permissions

Use symlink_permissions to assign ownership to symbolic links. 

symlink_permissions "/path/to/symlink" "user:group"
Example:

symlink_permissions "/usr/local/bin/my_symlink" "myuser:mygroup"

Assigns ownership to myuser:mygroup for the symbolic link.
