Service Management Script Documentation

Overview

This script provides a set of functions for managing system services in a Linux environment. It supports parallel execution for efficient management of multiple services.

Prerequisites

  • Bash shell
  • systemctl available on the system
  • Appropriate permissions to manage services

Setup

Save the script as service_utils.sh and make it executable: 

chmod +x service_utils.sh

Usage

Define Services

Edit the SERVICES array in the script to list the services you want to manage: 

SERVICES=("nginx" "mysql" "apache2")

Run the Script

Execute the script to perform batch operations on all services listed in the SERVICES array: 

./service_utils.sh

Available Functions

  • start_all_services: Starts all services in the SERVICES array.
  • stop_all_services: Stops all services in the SERVICES array.
  • restart_all_services: Restarts all services in the SERVICES array.
  • reload_all_services: Reloads the configuration for all services in the SERVICES array.
  • status_all_services: Checks the status of all services in the SERVICES array.
  • enable_all_services: Enables all services in the SERVICES array to start on boot.
  • disable_all_services: Disables all services in the SERVICES array from starting on boot.

Example

To start, restart, and check the status of services:

# Start all services
          start_all_services

          # Restart all services
          restart_all_services

          # Check status of all services
          status_all_services

Notes

  • Ensure the SERVICES array is updated with the services you want to manage.
  • The script supports multithreading, so operations on services run in parallel.
  • Use wait_for_background_tasks to ensure all background tasks are completed before proceeding.
