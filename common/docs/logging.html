Logging Utility Documentation

Overview

The logging.sh script provides logging capabilities for Bash scripts. It allows you to log informational messages, warnings, and errors with timestamped entries. Logs are displayed with color-coded messages for better readability in the terminal and are saved to a specified log file.

Features

  • Color-Coded Logs:
    • Info (Green): General information about script execution.
    • Warning (Yellow): Non-critical issues that require attention.
    • Error (Red): Critical issues that disrupt script execution.
  • Timestamped Entries: Each log entry includes the current date and time.
  • Log File Support: Messages are written to both the terminal and a specified log file.

Prerequisites

  • Ensure the logging.sh script is available and sourced in your environment.
  • Set the environment variable LOG_FILE to define where log entries should be saved.

Usage

The utility provides three logging functions:

log_info

Logs an informational message in green.

Example:

log_info "Backup process started."

Output:

[INFO] YYYY-MM-DD HH:MM:SS Backup process started.

log_warning

Logs a warning message in yellow.

Example:

log_warning "Source directory is empty."

Output:

[WARNING] YYYY-MM-DD HH:MM:SS Source directory is empty.

log_error

Logs an error message in red.

Example:

log_error "Failed to create backup."

Output:

[ERROR] YYYY-MM-DD HH:MM:SS Failed to create backup.

Integration

Step 1: Include the Script

To use the logging utility, source the logging.sh script in your Bash script: 

source /path/to/logging.sh

Step 2: Set the Log File

Set the LOG_FILE environment variable to define the log file path: 

export LOG_FILE="/path/to/log_file.log"

Step 3: Use Logging Functions

Call the appropriate logging functions to log messages:


      log_info "Starting script execution."
      log_warning "Input directory is missing, using default."
      log_error "Failed to connect to the database."

Log File Example

Entries in the specified log file will look like this:


      [INFO] YYYY-MM-DD HH:MM:SS Starting script execution.
      [WARNING] YYYY-MM-DD HH:MM:SS Input directory is missing, using default.
      [ERROR] YYYY-MM-DD HH:MM:SS Failed to connect to the database.

Best Practices

  • Define and export LOG_FILE before using the logging functions.
  • Use log_info for normal operations, log_warning for recoverable issues, and log_error for critical problems.
  • Periodically review the log file for troubleshooting and monitoring.
